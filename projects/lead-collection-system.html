📋 Lead Collection & Management System

A full-stack lead collection and management platform for marketing campaigns (for example Facebook Ads). It captures high-quality leads and reduces data loss by saving user input in real time, even when the form is never submitted.

  • 🔧 Tech Stack: React (Vite), Django, Django REST Framework, PostgreSQL (production) / SQLite (development)
  • 🌍 Live — Lead Form: lead-production-0b8b.up.railway.app
  • 📊 Live — Admin Dashboard: Dashboard on Railway
  • 🚀 Deployment: Railway (separate frontend and backend services)

🎯 Problem Statement

Typical lead flows lose incomplete submissions, discard users who abandon the form, and struggle with inconsistent data quality. This system addresses that with real-time draft saving, flexible validation (phone or email), and an admin experience built for non-technical operators—not only the default Django admin.

✨ Core Features

  • Real-time draft auto-save — Persists input while typing without requiring submit, reducing drop-off loss.
  • Smart submission rules — A lead counts as valid when phone or email is present.
  • Admin dashboard — Browse leads with filters for submission status (Draft / Submitted), contact type (Phone / Email), and activity date.
  • Soft delete & trash — deleted_at-style workflow: restore or permanently delete (password-protected).
  • CSV export — Filtered export as Excel-friendly UTF-8 with BOM; preserves values such as leading zeros in phone numbers.
  • Custom admin UI — Improved login and dedicated dashboards beyond stock Django admin.

🏗️ System Architecture

Frontend — React with Vite: lead form with auto-save; routes include / (form), /login, and /dashboard for admin flows.

Backend — Django + DRF exposing REST endpoints such as POST /api/save-draft/, POST /api/submit/, GET /api/leads/dashboard/, and GET /api/health/.

Lead lifecycle — Draft captures partial progress; Submitted applies validation. Activity tracking uses a last-activity timestamp for filtering and analytics. Active lists exclude soft-deleted rows; trash shows only deleted entries.

API security uses environment-based configuration with appropriate CORS and CSRF settings for cross-origin communication between Railway-hosted services.

📌 Use Cases & Outcomes

Suited for paid social landing pages, sales teams that need clean contact data, and any business that wants fewer “invisible” leads. Outcomes include higher effective capture rate, less permanent loss from partial forms, and clearer operational decisions from structured filters and exports.
