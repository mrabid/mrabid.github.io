📦 Chuti Group Inventory Management System

A full-stack inventory management web application built for Chuti Group to streamline internal inventory tracking, user management, and service request workflows. The system features a React frontend and Django REST API backend, deployed on Railway.com with CI/CD auto-deployment from GitHub.

  • 🔧 Tech Stack: React 18, Vite, Django 6.0, Django REST Framework, PostgreSQL
  • 🌍 Live Website: chutibd.com
  • 📌 Type: Full-Stack Web Application
  • 👨‍💻 Role: Full-Stack Developer
  • ☁️ Deployment: Railway.com (Auto-deploy from GitHub)

📋 Project Overview

The Chuti Group Inventory Management System is a comprehensive web application designed to manage internal inventory operations for Chuti Group. It provides a centralized platform where authorized users can track inventory items, manage user accounts, handle service requests, and view real-time data through an intuitive dashboard.

The application follows a modern decoupled architecture — the React frontend communicates with the Django REST API backend via RESTful endpoints, ensuring a clean separation of concerns and scalable design.

☁️ Deployment & CI/CD

The application is deployed on Railway.com as three separate services — React frontend, Django backend, and PostgreSQL database. The project is connected to a GitHub repository with automatic deployment enabled, meaning every push to the main branch triggers an automatic rebuild and redeploy of both frontend and backend services.

  • 🚀 Auto-deployment — Every git push to main triggers automatic redeploy on Railway
  • 🏗️ Multi-service architecture — Frontend, Backend, and Database run as separate Railway services
  • 🌐 Custom domain — Production app served at chutibd.com
  • 📦 Automated migrations — Database migrations run automatically on each deployment
  • 🗂️ Static file serving — WhiteNoise handles static files efficiently in production

🛠️ REST API Design

The backend exposes a well-structured RESTful API built with Django REST Framework, covering authentication, inventory operations, user management, and service requests. The API uses token-based authentication to secure all endpoints, ensuring only authorized users can access and modify data.
